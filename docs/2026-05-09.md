## 2. 架构修改历史

### 2.1 编码器重构（3路独立编码）

**修改前**（有问题）：
- `wind_encoder`: 先插值到 300x300 再卷积 → 引入插值噪声
- `terrain_encoder`: DEM + 梯度混在同一个 3ch 编码器 → 数值尺度冲突（DEM 0-724m，梯度 ±10）
- `roughness_encoder`: 正常

**修改后**（当前）：
- `wind_encoder`: 先在 9x9 原始分辨率卷积，再上采样 → 避免插值噪声
- `dem_encoder`: DEM 单独编码（标量场）
- `roughness_encoder`: 粗糙度单独编码
- 去掉显式梯度输入通道（ch4-5），让模型从 DEM 自己学坡度

**结论**: DEM 本身已包含坡度信息，显式梯度作为模型输入是多余的。但梯度在 **Loss 函数** 中作为物理约束（法向边界条件 u·n=0）是必需的。

### 2.2 物理约束层（PhysicsConstraintLayer）

接收 `output_mean`/`output_std` 进行物理空间操作：
- 地面 w 按高度衰减（linspace 0.1→1.0）
- k 用 softplus 强制非负

## 3. 训练问题

### 3.1 DataParallel 死锁

**现象**: 单进程卡死，GPU 显存占用但利用率为 0%，日志停在 "Parameters: 31.6M"

**根因**: `DataLoader(persistent_workers=True)` + `nn.DataParallel` 多进程冲突

**解决**: 弃用 DataParallel，改用 DDP (DistributedDataParallel)

### 3.2 DDP NCCL Watchdog Timeout

**现象**: 训练正常完成后（Early stopping），日志出现大量 NCCL timeout 报错

```
[Rank 1] Watchdog caught collective operation timeout: WorkNCCL(ALLREDUCE) ran for 600061ms
```

**根因**: rank 0 进程退出后，其他 rank 还在等待 allreduce 同步，10 分钟超时后崩溃

**影响**: 不影响训练结果，只是退出不干净

**解决方向**:
- 退出前调用 `dist.barrier()` + `cleanup_ddp()`
- 或设置 `NCCL_TIMEOUT` 环境变量

### 3.3 ProgressiveLossScheduler Stage 切换导致 Loss 暴涨

**现象**:
```
Epoch 30: Train 2.04, Val 2.65  (Stage 0, 权重较轻)
Epoch 31: Train 5.08, Val 4.74  (Stage 1, 物理约束权重↑)
Epoch 38: Train 2.25, Val 4.17  (Val Loss 持续不降，严重过拟合)
```

**Stage 0 权重**:
- mass_conservation: 0.01
- terrain_penalty: 0.01
- k_positive: 0.2

**Stage 1 权重**:
- mass_conservation: 0.05 (↑5x)
- terrain_penalty: 0.05 (↑5x)
- k_positive: 0.3 (↑1.5x)
- k_specialized: 0.7 (↑1.4x)

**根因**: Stage 1 物理约束权重跳涨 5 倍，模型还没学会数据拟合就被强制满足物理约束，导致：
1. Train Loss 先暴涨再缓慢下降
2. Val Loss 完全不降（泛化能力被破坏）

**解决方向**:
- 去掉 ProgressiveLossScheduler，用固定权重
- 或延长 Stage 0 到 50+ epoch
- 或 Stage 切换时平滑过渡（线性插值权重）

### 3.4 过拟合问题

从 Epoch 6 开始，Train Loss 持续下降但 Val Loss 停滞：
```
Epoch 6:  Train 3.21, Val 3.11  ← 差距小
Epoch 14: Train 2.42, Val 2.51  ← 差距 0.09 (最佳)
Epoch 30: Train 2.04, Val 2.65  ← 差距 0.61 (过拟合加重)
Epoch 38: Train 2.25, Val 4.17  ← 差距 1.92 (严重过拟合)
```

**根因**:
1. 模型 31M 参数对 10k 训练样本偏大
2. 物理约束权重在 Stage 1 后过大，模型被迫拟合训练集的噪声以满足约束
3. Dropout=0.1 可能不够

**解决方向**:
- 增加 Dropout (0.1 → 0.2-0.3)
- 增大 Weight Decay (0.01 → 0.05)
- 减小 base_channels (64 → 32)
- 数据增强（随机翻转、旋转）

## 4. 训练脚本问题

### 4.1 缺失 R² 指标

训练日志只有 Loss，没有 R²/RMSE。需要在验证循环中加入：
```python
from scipy.stats import pearsonr
r2 = pearsonr(pred.flatten(), target.flatten())[0] ** 2
```

### 4.2 DataLoader num_workers

`num_workers > 0` 在 DataParallel 下会死锁，`num_workers=0` 又太慢。DDP 下 `num_workers=0` 是安全的但数据加载成为瓶颈。

**建议**: DDP 下可以尝试 `num_workers=2` + `persistent_workers=False`
